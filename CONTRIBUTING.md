# Как вносить изменения

## 1. Перед началом работы

- Синхронизируйте ветку с актуальным `main`.
- Установите зависимости: `npm install`.
- Проверьте, что локальные `.env` не попадают в коммит.

## 2. Стандарт разработки

- Все изменения должны соответствовать ТЗ и приложениям заказчика.
- Новые допущения фиксируйте в `docs/assumptions.md`.
- Не добавляйте в Git артефакты сборки и локальные зависимости.

## 3. Проверки перед PR

```bash
npm run build:web
npm run build:server
npm run test -w apps/web
npm run test -w apps/server
```

Если есть изменения в БД:

```bash
npm run prisma -w apps/server -- migrate dev
npm run prisma -w apps/server -- generate
```

## 4. Правила по коммитам

- Один коммит — одна логическая задача.
- Сообщение коммита: коротко и по сути, на русском.
- В описании PR указывайте:
  - что изменено,
  - зачем изменено,
  - как проверяли.

## 5. Обновление документации

При любом изменении логики/ролей/навигации обновляйте:

- `docs/business_rules.md`
- `docs/rbac.md`
- `docs/navigation_map.md`
- `docs/tz_compliance_report.md` (если влияет на соответствие ТЗ)
