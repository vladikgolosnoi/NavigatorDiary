# Доменная модель

Ниже — ключевые сущности и связи между ними. Детальная схема реализована в Prisma.

Примечание по релизу: в клиентской навигации оставлены экраны из минимальной таблицы ТЗ; часть сервисных сущностей (например, заметки/обращения/уведомления) может присутствовать в бэкенде, но не выводится как отдельные пользовательские экраны.

## Пользователи и команды
- **User**: участник системы; поля: ФИО, дата рождения, контактные данные, статус подтверждения, роль.
- **Team**: команда 6–12 человек; поля: населённый пункт, образовательное учреждение, название.
- **TeamMembership**: связь пользователя с командой и ролью в ней (руководитель/навигатор).
- **Role**: Организатор/Администратор, Руководитель, Навигатор.

## Каталоги целей
- **AgeGroup**: возрастная группа (min/max возраст, порядок отображения).
- **GoalSphere**: сфера развития.
- **Competency**: компетентность внутри сферы.
- **Goal**: цель внутри компетентности с привязкой к возрастной группе.
- **GoalActivityLink**: ссылки на активности/материалы для достижения цели.

## Выбор и прогресс целей
- **GoalSelection**: набор выбранных целей пользователем (дата выбора, ограничение 12, следующая дата выбора).
- **UserGoal**: выбранная цель пользователя (статус, даты, комментарии).
- **GoalProgress**: отметки прогресса (не чаще 1 раза в неделю).
- **GoalReaction**: реакция/голос других навигаторов за цель.
- **GoalConfirmation**: подтверждение цели руководителем/организатором.

## Каталоги специальностей
- **Area**: область.
- **Specialty**: специальность в области.
- **SpecialtyLevel**: уровень (бронза/серебро/золото).
- **SpecialtyVideoLink**: ссылки на видео о специальностях.
- **SpecialtyMaterialLink**: ссылки на материалы.

## Выбор и прогресс специальностей
- **UserSpecialty**: выбранная активная специальность пользователя.
- **SpecialtyChecklistItem**: пункт чек‑листа уровня.
- **UserSpecialtyChecklist**: выполнение пунктов пользователем.
- **SpecialtyConfirmation**: подтверждение уровня (руководитель для бронзы, организатор для серебра/золота).

## Достижения
- **AchievementStatus**: статус по возрасту (Следопыт/Навигатор/Новатор/Странник).
- **AchievementStage**: этап по числу достигнутых целей (старт/путь/тропа/маршрут/экспедиция/успех).

## Хатка бобра
- **BranchAward**: начисление веточек организатором участникам команды (тип, сумма, дата).
- **BeaverHutTotals**: агрегированные значения жёлудей/веточек/поленьев (рассчитываются на основе целей, начислений и специальностей).

## Взаимодействия
- **ChatMessage**: сообщение командного чата.
- **ChatReaction**: реакция на сообщение.
- **Announcement**: анонс для главной страницы.
- **Notification**: уведомление от организатора (всем или конкретным командам).

## Личные данные и поддержка
- **AnonymousNote**: личные анонимные заметки (видны только владельцу).
- **Appeal**: обращение/тикет к организаторам.

## Аудит
- **AuditLog**: фиксирует подтверждения, голоса, изменения данных.
