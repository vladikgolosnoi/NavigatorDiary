# Чек-лист перед публикацией на GitHub

## 1) Чистота репозитория

- [ ] Нет `node_modules`, `dist`, `build`, `.env`, `.DS_Store`.
- [ ] `.gitignore` покрывает локальные и служебные файлы.
- [ ] В репозитории нет токенов, паролей, приватных ключей.

## 2) Документация

- [ ] `README.md` актуален и содержит запуск, стек, структуру, ссылки на docs.
- [ ] `docs/specs/` содержит исходные материалы ТЗ в одном месте.
- [ ] `docs/tz_compliance_report.md` актуален.
- [ ] `docs/user_guide_testing.md` описывает проверку по ролям.
- [ ] `docs/hr/portfolio_overview.md` заполнен для HR/техлида.

## 3) Визуальная презентация

- [ ] Добавлены 6–8 скриншотов в `docs/screenshots/`.
- [ ] В `README.md` отображаются скриншоты.

## 4) Техническая готовность

- [ ] Успешно: `npm run build:web`
- [ ] Успешно: `npm run build:server`
- [ ] Успешно: `npm run test -w apps/web`
- [ ] Успешно: `npm run test -w apps/server`

## 5) Коммиты

- [ ] Сообщения коммитов осмысленные и краткие.
- [ ] Нет «мусорных» коммитов вида `fix`, `tmp`, `wip`.
- [ ] Изменения сгруппированы по смыслу (структура, функциональность, документация).

## 6) Чистый старт для нового публичного репозитория (рекомендуется)

Если локальная история разработки содержит лишние артефакты, публикуйте проект с новой чистой историей:

```bash
cd "<путь_к_проекту>"
rm -rf .git
git init
git add .
git commit -m "Инициализация проекта: Дневник навигатора"
```

Далее создайте пустой репозиторий на GitHub и привяжите `origin`.
