# Архитектура

## Обзор
«Дневник навигатора» — прогрессивное web‑приложение (PWA) для участников проекта «Дружина навигаторов». Решение построено как monorepo с npm workspaces и разделено на клиент, сервер и общий пакет типов.

## Структура репозитория
- `apps/web` — PWA (React + Vite + TypeScript)
- `apps/server` — API (NestJS + Prisma + PostgreSQL)
- `packages/shared` — общие типы/DTO

## Компоненты
- **Web‑клиент**: интерфейсы для навигаторов, руководителей и организаторов; маршрутизация, локальное сохранение последнего экрана/состояния.
- **API‑сервер**: REST + WebSocket; аутентификация (JWT), RBAC, бизнес‑правила, чат, уведомления, анонсы, аудит.
- **База данных**: PostgreSQL, доступ только сервером.
- **Shared**: типы и DTO для согласованности клиента и сервера.

## Потоки данных
1. Клиент отправляет запросы в API с JWT.
2. Сервер валидирует роль и бизнес‑правила.
3. Сервер читает/пишет данные в PostgreSQL.
4. Сервер публикует события в чат и формирует уведомления.

## Безопасность и соответствие требованиям
- Все данные хранятся и обрабатываются на стороне сервера.
- Нет внешних сервисов/интеграций.
- RBAC‑ограничения на уровне гардов/политик.
- Дружелюбные сообщения об ошибках.
- Возврат пользователя к последнему состоянию (localStorage).

## Масштабирование и разделение ответственности
- Модули сервера разделены по доменам: Auth, Teams, Goals, Specialties, Achievements, BeaverHut, Chat, Announcements, Notifications, Notes, Appeals, Audit.
- Общие типы вынесены в `packages/shared` для согласованности.
