## 1) Минимальная таблица экранов (ТЗ)

Покрытие экранов:

1. Авторизация / регистрация команды — реализовано (`/auth/team`)
2. Авторизация / регистрация пользователя — реализовано (`/auth/user`)
3. Главная страница (логотипы, приветствие, объявления) — реализовано (`/home`)
4. Личный кабинет (данные, смена пароля, переходы) — реализовано (`/profile`)
5. Каталог целей (возраст → сфера → компетентность → цель, до 12) — реализовано (`/goals/catalog`)
6. Каталог специальностей (область, видео, специальность, уровень) — реализовано (`/specialties/catalog`)
7. Мои цели (прогресс не чаще 1 раза в неделю, активности, комментарии) — реализовано (`/goals/my`)
8. Мои специальности (чек‑лист прогресса + материалы) — реализовано (`/specialties/my`)
9. Мои достижения (статус по возрасту + этапы) — реализовано (`/achievements`)
10. Хатка бобра (жёлуди/веточки/поленья) — реализовано (`/beaver-hut`)
11. Чат команды (сообщения, реакции, автособытия) — реализовано (`/chat`)
12. Дополнительный экран (условия, конфиденциальность, консультант) — реализовано (`/extra`)

## 2) Данные из приложений

### Приложение 2 (каталог целей)
- Импортирована структура: возрастная группа → сфера → компетентность → цель.
- В БД после сида: `AgeGroup=7`, `Sphere=6`, `Competency=21`, `Goal=343`.

### Приложение 3 (специальности)
- Импортированы все 10 специальностей, 3 уровня и чек‑листы.
- В БД после сида: `Area=1`, `Specialty=10`, `SpecialtyLevel=30`, `SpecialtyChecklistItem=246`.

### Значки (`docs/specs/05_appendix_badges.pdf`)
- Добавлены и сопоставлены:
  - маскоты: `бобер`, `феникс`;
  - этапы: `старт`, `путь`, `тропа`, `маршрут`, `экспедиция`, `успех`.
- Используются на главной и в достижениях.

### Материалы и ссылки
- Убраны ссылочные заглушки `example.com`.
- Материалы целей и специальностей ведут на рабочие файлы в `apps/web/public/resources`.
- Видеоссылки по специальностям ведут на реальную поисковую выдачу YouTube по названию специальности.

## 3) Роли и навигация

- Навигатор: в верхнем/мобильном меню только ключевые вкладки (без лишних админских разделов).
- Руководитель: отдельный дашборд (`/leader`) без заглушек.
- Организатор: отдельный дашборд (`/organizer`) без заглушек.
- Маршруты ограничены по ролям через `RequireRole`.
- В верхнем меню добавлен отдельный пункт `Хатка бобра` для ролей, которым он нужен по ТЗ.

## 4) Что убрано как лишнее для финальной версии

- Из пользовательской навигации убраны экраны вне минимальной таблицы ТЗ:
  - `Уведомления`,
  - `Анонимные заметки`,
  - `Обращения`.
- В панели организатора убраны блоки, не требуемые минимальным списком экранов (оставлены подтверждения и анонсы; начисление веточек — через `Хатку бобра`).

## 5) Проверка качества

- Сборка фронта: успешно (`npm run build -w apps/web`).
- Сборка сервера: успешно (`npm run build -w apps/server`).
- Миграции + сид: успешно (Docker, Prisma migrate/seed).
- Авторизация демо‑аккаунтов: успешно.
- Онлайн-консультант в `/extra` работает через реальные обращения (`/appeals`): создание, ответы, закрытие.
- В панели организатора реализована обработка обращений консультанта без заглушек.
- «Хатка бобра»: жёлуди начисляются по этапам из ТЗ (`6/24/48/72/144/216`), веточки и поленья — по правилам ТЗ.
