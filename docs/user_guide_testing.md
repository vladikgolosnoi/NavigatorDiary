# Инструкция для пользователя и тестирования

Дата: 03.03.2026

Материалы заказчика: `docs/specs/`

## 1) Как запустить проект

### Backend + БД (Docker)
```bash
docker compose -p navigator-diary up -d --build
docker compose -p navigator-diary exec -T server sh -lc 'npx prisma migrate deploy && npx prisma db seed'
```

### Frontend (Vite)
```bash
npm run dev -w apps/web -- --host 0.0.0.0 --port 5173
```

После запуска:
- веб: `http://localhost:5173`
- API: `http://localhost:3000/api`

Для телефона (в той же сети Wi‑Fi):  
`http://<локальный-ip-компьютера>:5173`

## 2) Демо‑аккаунты

- Организатор: `organizer@demo.local` / `Demo123!`
- Руководитель: `leader@demo.local` / `Demo123!`
- Навигатор: `navigator@demo.local` / `Demo123!`

## 3) Что проверять по ролям

### Навигатор
1. Войти как `navigator@demo.local`.
2. Открыть:
   - `Мои цели` (`/goals/my`) — комментарий, прогресс, реакция на цели команды.
   - `Мои специальности` (`/specialties/my`) — чек‑лист и подтверждение.
   - `Мои достижения` (`/achievements`) — возрастной статус, этап, значки.
   - `Хатка бобра` (`/beaver-hut`) — жёлуди/веточки/поленья.
   - `Чат` (`/chat`) — сообщения и реакции.
3. Проверить мобильную навигацию внизу экрана (на телефоне).
4. В `Дополнительно` (`/extra`) отправить сообщение онлайн-консультанту (создаётся обращение).

### Руководитель
1. Войти как `leader@demo.local`.
2. Открыть `Руководитель` (`/leader`).
3. Проверить:
   - подтверждение пользователей команды,
   - подтверждение целей,
   - подтверждение специальностей (бронза).
4. Проверить, что вкладки навигатора недоступны.

### Организатор
1. Войти как `organizer@demo.local`.
2. Открыть `Организатор` (`/organizer`).
3. Проверить:
   - подтверждение команд,
   - подтверждение пользователей,
   - подтверждение целей/специальностей,
   - управление анонсами,
   - обработку обращений онлайн-консультанта (ответ/закрытие).
4. Открыть `Хатка бобра` (`/beaver-hut`) и проверить начисление веточек команде.

## 4) Быстрый чек мобильной адаптации

На ширине телефона проверить:
- верхний блок без горизонтальных "ломаний";
- нижнюю мобильную навигацию (иконки + подписи);
- читаемость карточек и кнопок;
- отсутствие горизонтального скролла у основных экранов.

## 5) Проверка экрана «Авторизация команды»

1. Открыть `/auth/team`.
2. В режиме «Вход команды» ввести название активной команды (`Команда Демонстрация`).
3. Проверить, что система находит команду и переводит на вход пользователя.
4. В режиме «Регистрация команды» отправить тестовую заявку и проверить появление статуса ожидания.

## 6) Если вход не работает

1. Обновить страницу с очисткой кеша (`Cmd+Shift+R`).
2. Убедиться, что API доступен: `http://localhost:3000/api`.
3. Проверить, что веб запущен на `5173` (`npm run dev -w apps/web`).
4. Если открыто как PWA — удалить приложение/данные сайта и открыть снова.
